How to Test the Issue:

  1. Open your SAGE application in a browser
  2. Press F12 to open Developer Console
  3. Click on the Console tab
  4. Navigate to the Content tab in your app
  5. Type this exact text in the textarea:
    write a letter from Santa Clause to Donald Trump
  6. Click the Generate Content button
  7. Check the console for these debug messages
Expected Console Output:
[Home] userPrompt changed to: write a letter from Santa Clause to Donald Trump
[Home] userPrompt length: 48
[UserPromptPanel] Generate button clicked
[UserPromptPanel] Current userPrompt: write a letter from Santa Clause to Donald Trump
[UserPromptPanel] userPrompt length: 48
[Debug] handleGenerate called with:
- customPrompt: undefined
- userPrompt state: write a letter from Santa Clause to Donald Trump
- promptToUse: write a letter from Santa Clause to Donald Trump
- promptToUse length: 48
If You See This Instead (The Bug):
[UserPromptPanel] Current userPrompt: Generate content
[UserPromptPanel] userPrompt length: 16
- promptToUse: Generate content
- promptToUse length: 16

This would indicate the state is being overwritten somewhere.

What to Report Back:
  • Copy the console output you see
  • Note if the Santa letter was generated correctly or not
  • Check if anything happened between typing and clicking (like a popup or dialog)
